# The Standard Team — Commits — Checklist

- [ ] For TDD categories: failing test committed as `[Test Name] -> FAIL` (tst-commits-001)
- [ ] For TDD categories: passing implementation committed as `[Test Name] -> PASS` (tst-commits-002)
- [ ] TDD commits do not use the `[CATEGORY]: [Description]` format (tst-commits-003)
- [ ] Non-TDD commits use the format `[CATEGORY]: [Description Of Work Completed]` (tst-commits-004)
- [ ] `[CATEGORY]` is in CAPS (tst-commits-005, tst-commits-007)
- [ ] `[CATEGORY]` is from the approved category list (tst-commits-005)
- [ ] `[Description]` is in Pascal Case (tst-commits-006)
- [ ] `[Description]` is not vague (tst-commits-008)
- [ ] TDD category work uses FAIL/PASS format — not CATEGORY: Description (tst-commits-009)
- [ ] Non-TDD category work uses CATEGORY: Description format (tst-commits-010)
