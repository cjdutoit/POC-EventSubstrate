# The Standard — Core Principles — Checklist

- [ ] Every class has one and only one responsibility (ts-core-001)
- [ ] Every class, method, and variable is named by what it is, not how it works (ts-core-002)
- [ ] No layer calls more than one layer below itself (ts-core-003)
- [ ] Every public member can be tested without modification (ts-core-004)
- [ ] No internal implementation details are exposed through public members (ts-core-005)
- [ ] No business rules exist in controllers or brokers (ts-core-006)
- [ ] No comments are required to understand any method or variable name (ts-core-007)
- [ ] No class is large enough that it clearly handles more than one concern (ts-core-008)
- [ ] Volatile logic is separated from stable logic (ts-core-009)
