# The Standard — Practices — Checklist

- [ ] Work was done on a fork — no direct pushes to upstream (ts-practices-001)
- [ ] Branch name follows `users/{handle}/{type}/{short-description}` in lowercase kebab-case (ts-practices-002)
- [ ] All commit messages use ALL CAPS category prefix and imperative mood (ts-practices-003)
- [ ] No commit message uses vague words: fix, update, changes, wip (ts-practices-003b)
- [ ] PR is focused on a single concern (ts-practices-004)
- [ ] Build passes and all tests are green (ts-practices-005)
- [ ] No secrets, connection strings, or API keys are present in source code (ts-practices-006)
- [ ] CI pipeline is configured using ADotNet or YAML-based tooling (ts-practices-007)
- [ ] PR title follows `[CATEGORY]: [Description Of Work Completed]` format
- [ ] PR description includes `Closes #[issue-number]` where applicable
