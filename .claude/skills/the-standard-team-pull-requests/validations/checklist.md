# The Standard Team — Pull Requests — Checklist

- [ ] PR title follows the format `[CATEGORY]: [Description Of Work Completed]` (tst-pull-requests-001)
- [ ] `[CATEGORY]` is in CAPS (tst-pull-requests-002, tst-pull-requests-008)
- [ ] `[CATEGORY]` is from the approved category list (tst-pull-requests-002, tst-pull-requests-004, tst-pull-requests-010)
- [ ] `[Description]` is in Pascal Case (tst-pull-requests-003)
- [ ] `[Description]` is not vague (tst-pull-requests-009)
- [ ] Issues that should be closed on merge use `Closes #[issue-number]` in the description (tst-pull-requests-005)
- [ ] Multiple issues use `Closes #10, closes #123` format (tst-pull-requests-006)
- [ ] Issues linked without auto-close use `#[issue-number]` without keyword (tst-pull-requests-007)
