# PR Checklist

- [ ] PR title follows `[CATEGORY]: [Description Of Work Completed]` format
- [ ] Category is in CAPS from the approved list
- [ ] Description is in Pascal Case
- [ ] Description is specific — not vague
- [ ] Related issues are linked with `Closes #[issue-number]` or `#[issue-number]`
- [ ] Build is passing
- [ ] All tests are green
- [ ] PR is focused on a single concern
