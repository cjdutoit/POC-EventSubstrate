# The Standard Team — Branching — Checklist

- [ ] Repository was forked before contributing — not cloned directly from upstream (tst-branching-001)
- [ ] Branch was created on the personal fork — not on the upstream repository (tst-branching-003)
- [ ] Branch name follows the pattern `users/[username]/[category]-[entity]-[action]` (tst-branching-004)
- [ ] `[username]` is the contributor's GitHub username (tst-branching-005)
- [ ] `[category]` is taken from the approved category list (tst-branching-006)
- [ ] `[entity]` identifies the domain entity (tst-branching-007)
- [ ] `[action]` describes the action performed (tst-branching-008)
- [ ] All branch name segments are lowercase and hyphen-separated (tst-branching-009)
- [ ] Branch is focused on a single concern (tst-branching-010)
- [ ] Branch name is not a forbidden generic name: fix, test, temp, wip (tst-branching-012)
