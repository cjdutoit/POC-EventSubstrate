# The Standard React TypeScript — Brokers — Checklist

- [ ] Brokers are the only layer talking to external dependencies (tsr-brokers-001)
- [ ] All broker methods use async/await for I/O (tsr-brokers-002)
- [ ] External exceptions wrapped in broker exceptions (tsr-brokers-003)
- [ ] API methods use standard HTTP verbs (tsr-brokers-004)
- [ ] Method names follow `{verb}{Entity}Async` pattern (tsr-brokers-005)
- [ ] Storage brokers abstract localStorage/sessionStorage (tsr-brokers-006)
- [ ] Logging brokers abstract console or external logging (tsr-brokers-007)
- [ ] No business logic in brokers (tsr-brokers-008)
- [ ] Brokers injected via dependency injection (tsr-brokers-009)
- [ ] DateTime brokers abstract Date operations (tsr-brokers-010)
- [ ] Brokers organized by type (apis, storages, loggings, datetimes) (tsr-brokers-011)
- [ ] Class names follow `{Entity}{Type}Broker` pattern (tsr-brokers-012)
- [ ] Each broker serves one external dependency (tsr-brokers-013)
- [ ] Brokers expose interfaces for testing (tsr-brokers-014)
- [ ] Original exception preserved as inner exception (tsr-brokers-016)
- [ ] No swallowed exceptions (tsr-brokers-017)
- [ ] No error codes — use exceptions (tsr-brokers-018)
- [ ] Brokers don't call other brokers (tsr-brokers-019)
- [ ] No retries or circuit breaking in brokers (tsr-brokers-020)
- [ ] API brokers accept AbortSignal for cancellation (tsr-brokers-021)
