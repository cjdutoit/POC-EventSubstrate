# The Standard React TypeScript — Foundation Services — Checklist

- [ ] Service wraps exactly one broker (tsr-services-001)
- [ ] All inputs validated before broker calls (tsr-services-002)
- [ ] Broker exceptions mapped to service exceptions (tsr-services-003)
- [ ] Try-catch pattern used for broker calls (tsr-services-004)
- [ ] Broker injected via dependency injection (tsr-services-005)
- [ ] Method names mirror broker methods appropriately (tsr-services-006)
- [ ] TryCatch wrapper implemented (tsr-services-007)
- [ ] No UI logic in service (tsr-services-008)
- [ ] Service doesn't call other services (tsr-services-009)
- [ ] Exception types defined (ValidationException, DependencyException, ServiceException) (tsr-services-010)
- [ ] Null/undefined validation present (tsr-services-011)
- [ ] Required string fields validated (tsr-services-012)
- [ ] ID fields validated for retrieval operations (tsr-services-013)
- [ ] Structural validation present where appropriate (tsr-services-014)
- [ ] Validation exceptions include field name and reason (tsr-services-015)
- [ ] Validation happens before broker calls (tsr-services-016)
- [ ] Broker exceptions wrapped in DependencyException (tsr-services-017)
- [ ] Validation failures throw ValidationException (tsr-services-018)
- [ ] Service errors wrapped in ServiceException (tsr-services-019)
- [ ] Inner exceptions preserved (tsr-services-020)
- [ ] Exception messages are user-facing (tsr-services-021)
- [ ] Service in correct directory (services/foundations/{domain}/) (tsr-services-022)
- [ ] Class name follows pattern (tsr-services-023)
- [ ] Interface defined (tsr-services-024)
- [ ] One service per file (tsr-services-026)
