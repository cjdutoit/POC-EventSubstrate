# The Standard React TypeScript — View Services — Checklist

- [ ] View service transforms foundation models to view models (tsr-view-001)
- [ ] Foundation service injected, not broker (tsr-view-002)
- [ ] No business logic in view service (tsr-view-003)
- [ ] UI transformations handled (tsr-view-004)
- [ ] Exceptions mapped to view layer (tsr-view-005)
- [ ] Returns view models only (tsr-view-006)
- [ ] Foundation models not modified (tsr-view-007)
- [ ] UI logic centralized (tsr-view-008)
- [ ] Dates formatted for display (tsr-view-009)
- [ ] Status codes mapped to labels (tsr-view-010)
- [ ] Computed fields calculated here (tsr-view-011)
- [ ] ClassNames generated here (tsr-view-012)
- [ ] In correct directory (tsr-view-014)
- [ ] Class name follows pattern (tsr-view-015)
- [ ] Interface defined (tsr-view-016)
- [ ] One service per file (tsr-view-018)
- [ ] Doesn't call other view services (tsr-view-019)
- [ ] Handles null data gracefully (tsr-view-020)
