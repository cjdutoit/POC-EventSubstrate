# The Standard C# — Methods — Checklist

- [ ] Method name contains a verb (tsc-csharp-methods-001)
- [ ] Async method name ends with `Async` (tsc-csharp-methods-002)
- [ ] Parameters are named with entity prefix, not generic names (tsc-csharp-methods-003, tsc-csharp-methods-004)
- [ ] No unnamed literal values in method calls (tsc-csharp-methods-005)
- [ ] One-liner methods use fat-arrow syntax (tsc-csharp-methods-006)
- [ ] Fat-arrow lines over 120 chars break after `=>` with extra tab indent (tsc-csharp-methods-007)
- [ ] Multi-liner methods use scoped block `{ }` (tsc-csharp-methods-008)
- [ ] `return` in multi-liner method is preceded by a blank line (tsc-csharp-methods-009)
- [ ] Consecutive calls follow stacking/blank-line rules (tsc-csharp-methods-010)
- [ ] Method declarations over 120 chars break to one-param-per-line (tsc-csharp-methods-011)
- [ ] Method chains use uglification indentation (tsc-csharp-methods-012)
